Variance Analysis

The ~30% variance between the two Focus/Shadow runs is attributable to:

1. LLM Response Variability EXPECTED

GPT models produce different outputs even with identical prompts. The query decomposer may generate different sub-query structures, leading to different retrieval patterns and token usage.

2. Cache State Differences EXPECTED

The first run (Method 3) may have benefited from warmer caches if run immediately after system initialization. Retrieval cache hit rates can vary based on prior queries.

3. Prompt 7 Outlier NOTABLE

The final prompt ("Summarize this conversation with 6 bullets per topic") shows significant variance:

  • Method 3: 17,316 tokens, $0.0143, 33.0s
  • Method 2: 37,111 tokens, $0.0410, 1m 43s

This 2x difference in the summary prompt accounts for most of the overall variance and likely reflects different conversation state accumulation.

Key Finding

Despite the variance, both Focus/Shadow runs achieved 70%+ cost reduction vs Direct Chat, validating the RLM pipeline's consistent efficiency gains. The variance is within acceptable bounds for production use.
