Important Clarification: What This Test Measures

This test measures response capability — whether the system can reference and use prior conversation context. It does NOT measure the factual accuracy of response content against source documents.

  • RLM producing a response does not mean 100% accuracy — humans must verify content
  • Direct Chat failing to respond = context window limitation (not hallucination)
  • Next step: Human A/B testing to verify factual accuracy

2. Grading Methodology & Limitations

What "Response Capability" Means

Each prompt was evaluated on whether the system could successfully reference and respond to memory-dependent questions. The grading criteria:

Category Criteria Example
Capable System references prior response and provides substantive answer "The FIRST decision listed (in response #3): 'Maintain the federal funds target range at 5¼–5½%.'"
Not Capable System explicitly states it cannot access prior response "The meeting data I have does not include 'response #4'"
Partial System attempts answer but hedges or uses source data instead of memory "Based on the meeting data (not your prior response)..."

Critical Limitation: Accuracy Not Verified

When RLM states "The FIRST decision in response #3 was X" — this test verified that RLM attempted to recall response #3, but did NOT verify whether "X" is factually correct against the original source documents.

This is the gap that human A/B testing must fill.

What "Response Capability Rate" Actually Represents

  • Direct Chat 18%: Only 18% of memory-dependent prompts got a substantive answer (rest said "I don't have access to...")
  • RLM 96%: 96% of memory-dependent prompts got a substantive answer that referenced prior responses
  • Neither rate measures accuracy — only whether the system could engage with the question

6. Peer Review Findings

Issues Identified in Prior Reports

1. Terminology Clarification Needed

Prior reports used "memory recall rate" which could be misinterpreted as "factual accuracy rate." The correct term is "response capability rate" — measuring whether the system can engage with memory-dependent questions, not whether answers are factually correct.

2. "Cost-per-Successful-Response" Caveat

This metric is based on response capability, not verified accuracy. If human testing reveals RLM's accuracy is lower than expected, the cost-effectiveness calculation would change. Treat as preliminary until human verification.

3. "Catastrophic Failure" Terminology

While Direct Chat's context window limitation is severe, "catastrophic failure" may overstate the issue. More accurate: "context window exhaustion" — the model loses access to early conversation turns due to architectural limitations, not system failure.

4. Data Integrity Verified

All quantitative metrics (tokens, costs, response times, error counts) in prior reports match the underlying CSV data files. No inflation or embellishment of raw metrics detected.

Corrected Claims

Original Claim Corrected Statement
"RLM achieves 96% memory recall accuracy" RLM successfully responds to 96% of memory-dependent prompts. Accuracy of content requires human verification.
"Direct Chat fails catastrophically by Turn 9-10" Direct Chat experiences context window exhaustion by Turn 7-10, resulting in explicit inability to reference prior responses.
"RLM is 2.8x more cost-effective" RLM provides 2.8x more capable responses per dollar. Cost-effectiveness of accurate responses pending human verification.
"Zero hallucinations" Neither system fabricated information. Direct Chat's failures were due to memory loss, not hallucination. RLM's content accuracy not yet verified.

7. What This Test Proves (and Doesn't)

What This Test PROVES

  • RLM maintains conversation context through 100+ turns where Direct Chat cannot
  • RLM can reference specific prior responses by number (e.g., "response #3", "response #51")
  • RLM correctly identifies trap questions (items that don't exist in prior responses)
  • Direct Chat loses context by Turn 7-10 regardless of total conversation length
  • RLM's memory system works — it retrieves and uses prior conversation data

What This Test DOES NOT Prove

  • Factual accuracy of RLM responses against source documents
  • Completeness of RLM's recall (did it miss anything?)
  • Quality of synthesized answers vs. ground truth
  • Whether RLM leverages source data optimally
  • User satisfaction with response quality

The Gap: Response Capability vs. Accuracy

Consider this example from Turn 22:

Prompt: "In response #5, how many risks did you categorize as 'Critical'?" RLM Response: "Count of 'Critical' risks in response #5: 3"

This test verified: RLM referenced response #5 and provided a count.

This test did NOT verify: Whether there were actually 3 Critical risks, or whether RLM missed any, or whether the categorization was correct.

Human A/B testing needed: A human must review response #5 and confirm: "Yes, there were exactly 3 Critical risks."

8. Human A/B Testing Recommendations

Recommended A/B Test Protocol

Phase 1: Ground Truth Establishment

  1. For each of the 7 FOMC meeting agents, create a verified "answer key"
    • Topics discussed (verified list)
    • Decision-makers mentioned (verified list with roles)
    • Decisions made (verified list)
    • Action items assigned (verified list with owners)
    • Risks categorized (verified list with severities)
  2. This becomes the "ground truth" for accuracy scoring

Phase 2: Response Accuracy Scoring

Metric Definition Target
Precision Of items mentioned, how many are correct? >90%
Recall Of correct items, how many were mentioned? >80%
F1 Score Harmonic mean of precision and recall >85%
Hallucination Rate Items mentioned that don't exist in source <5%

Phase 3: Sample Selection

  • Select 30 representative prompts from the 100-question test
  • Include: 10 foundation prompts, 10 memory-recall prompts, 10 synthesis prompts
  • For each prompt, have 2 independent human raters score RLM's response
  • Calculate inter-rater reliability (target: Cohen's kappa >0.8)

Phase 4: Identify Improvement Areas

  • Which source data fields does RLM leverage well?
  • Which source data fields does RLM miss or underutilize?
  • Are there systematic errors (e.g., always missing action item owners)?
  • Does accuracy degrade with conversation length?

Expected Outcomes

Based on the response patterns observed, we hypothesize:

  • RLM Precision: Likely high (>85%) — RLM doesn't appear to fabricate data
  • RLM Recall: Unknown — may miss some details from source documents
  • Areas for Improvement: Likely in how well RLM extracts and synthesizes specific details from source agents

9. Conclusions & Next Steps

Validated Conclusion

RLM successfully solves the context window limitation problem.

For conversations requiring memory beyond 5-7 turns, RLM is demonstrably superior to Direct Chat in its ability to reference and use prior conversation context. This is a significant technical achievement.

Outstanding Question

How accurate is RLM's content?

This test proves RLM can reference prior responses. It does NOT prove that RLM's synthesized answers are factually correct against source documents. Human A/B testing is required to answer this question.

Immediate Next Steps

  1. Human A/B Testing (Priority 1)
    • Establish ground truth for 7 FOMC meeting agents
    • Score 30 sample responses for precision/recall/F1
    • Identify systematic accuracy gaps
  2. Source Data Leverage Analysis (Priority 2)
    • Analyze which agent fields RLM uses effectively
    • Identify underutilized data that could improve responses
    • Feed findings back to RLM prompt engineering
  3. Societies of Thought Integration (In Development)
    • Already in development by separate engineering team
    • Expected to improve synthesis quality for complex queries
    • Should be tested with same methodology after deployment
